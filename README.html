Complete Guide: Setting Up an EC2 Instance and Pointing a Namecheap Domain to It

This guide provides a full process for launching an Amazon EC2 instance, setting up a web server, and configuring a Namecheap domain to point to that instance.

Prerequisites

  • An AWS account
  • A registered domain on Namecheap
  • Basic knowledge of web servers and DNS settings

Part 1: Launching and Configuring an EC2 Instance

  1. Log into Your AWS Account:
    • Go to the AWS Management Console.
  2. Navigate to EC2:
    • In the Services menu, select EC2.
  3. Launch an Instance:
    • Click on the Launch Instance button.
  4. Choose an Amazon Machine Image (AMI):
    • Select an AMI suited to your needs (e.g., Amazon Linux, Ubuntu, etc.).
  5. Choose an Instance Type:
    • Select an instance type (e.g., t2.micro, which is eligible for the free tier) and click Next: Configure Instance Details.
  6. Configure Instance Details:
    • Adjust settings as needed (the default should be sufficient for basic setups). Click Next: Add Storage.
  7. Add Storage:
    • Adjust the size if necessary, then click Next: Add Tags.
  8. Add Tags (optional):
    • Add tags for organization (e.g., Name: MyWebServer). Click Next: Configure Security Group.
  9. Configure Security Group:
    • Create a security group to allow HTTP and SSH access:
      • Type: HTTP, Port: 80, Source: Anywhere (0.0.0.0/0)
      • Type: SSH, Port: 22, Source: Your IP (for security) or Anywhere (0.0.0.0/0) for broad access
    • Click Review and Launch.
  10. Launch the Instance:
    • Review your settings and click Launch.
    • Select an existing key pair or create a new one to use for SSH access. Download the private key file (*.pem) if creating a new pair.
  11. Access Your EC2 Instance:
    • Open a terminal (or PuTTY if using Windows) and run: 
      ssh -i /path/to/your-key.pem ec2-user@your-instance-public-ip
    • Replace /path/to/your-key.pem with the path to your downloaded key and your-instance-public-ip with the public IP.
  12. Install a Web Server:
    • For example, to install Apache on Amazon Linux: 
      sudo yum update -y
sudo yum install httpd -y
sudo systemctl start httpd
sudo systemctl enable httpd
    • Check that your web server is running by visiting your instance's public IP in a web browser.

Part 2: Configuring Your Domain on Namecheap

  1. Log into Your Namecheap Account:
    • Go to the Namecheap login page and log in.
  2. Navigate to Domain List:
    • Click on Domain List in the left sidebar.
    • Locate the domain you want to configure.
  3. Manage the Domain:
    • Click the Manage button next to your domain name.
  4. Change to BasicDNS (if applicable):
    • Find the Nameservers section. If it’s not already set, change to Namecheap BasicDNS and save.
  5. Access Advanced DNS:
    • Click on the Advanced DNS tab.
  6. Add A Records:
    • For the Root Domain (example.com):
      • Click on Add New Record.
      • Type: A Record
      • Host: @
      • Value: Your EC2 instance's public IP (e.g., 203.0.113.25)
      • TTL: Automatic
    • For the www Subdomain (www.example.com):
      • Click on Add New Record.
      • Type: A Record
      • Host: www
      • Value: Your EC2 instance's public IP (same as above)
      • TTL: Automatic
  7. Save Changes:
    • Ensure all records are saved.

Part 3: Testing the Configuration

  1. Wait for DNS Propagation:
    • DNS changes may take a few minutes to 48 hours to propagate. Use tools like WhatsMyDNS to verify.
  2. Test Your Domain:
    • After waiting, navigate to http://example.com and http://www.example.com in a web browser.
    • Confirm that your website displays correctly.

Conclusion

By following these steps, you’ve successfully spun up an EC2 instance, set up a web server, and configured your Namecheap domain to point to that instance. If any issues arise during the process, double-check the configurations and ensure your EC2 instance is running properly. If you have any questions, feel free to ask!
